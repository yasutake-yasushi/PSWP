# PSWP 共有用説明台本（3分版 / 15分版）

## 3分版（要点共有）

みなさんお疲れさまです。PSWPのCIとデプロイ周りを、まず運用に乗せるところまで整備しました。  
今日は3点だけ共有します。

### 1. CIログの見やすさ改善

Jenkinsで出ていたANSI制御文字のノイズを抑制しました。  
加えて、フロントのテスト警告を減らし、ログを回帰判定に使いやすくしました。  
狙いは、失敗時に本当に見るべき差分だけをすぐ見つけることです。

### 2. Jenkinsエージェントの自動起動化

Windows側でエージェントを自動起動できるようにしました。  
これで手動起動の抜け漏れを減らし、朝一や再起動後でもジョブを流しやすくなります。

### 3. 本番デプロイの土台作成

バックエンドはIIS配備用スクリプトと本番設定を追加。  
フロントは静的配信向けスクリプトとSPAルーティング設定を追加。  
デプロイ手順はDeploy配下にまとめ、引き継ぎしやすくしました。  
また、将来的なSQL Server利用を前提に、設定の切り替え方も入れています。

以上です。次は、実サーバ値を入れた最終パラメータ確定と、1回通しのリハーサルを実施すれば本番投入準備に入れます。

## 15分版（背景と詳細共有）

今日は、PSWPのCI運用とデプロイ運用を実務で回せる状態にするための変更を共有します。  
話す順番は、背景、実施内容、運用方法、残課題です。

### 1. 背景と課題（約2分）

これまでの課題は大きく3つでした。  
1つ目は、Jenkinsログにノイズが多く、失敗要因の特定に時間がかかること。  
2つ目は、フロントテストの警告で回帰判定がぶれやすいこと。  
3つ目は、エージェント起動や配備手順が人依存で再現性が弱いこと。  
今回はこの3点を優先して整備しました。

### 2. 実施内容（約6分）

まずCIです。  
Jenkins側でANSI由来のログノイズを抑え、読みやすさを改善しました。  
次にフロントテストです。  
警告を減らすために、非同期更新の扱いとテストの待機ポイントを整理し、回帰テストとして信頼できる出力に寄せました。  
結果として、ログ確認の初動がかなり速くなります。

次にエージェント運用です。  
Windowsで自動起動できる形を整備し、手動介入を減らしました。  
これにより、サーバ再起動後でもジョブ再開までの時間を短縮できます。

最後にデプロイです。  
バックエンド向けにはIIS配備スクリプト、Production設定、ホスティング設定を追加しました。  
フロント向けには、ビルド後アセットのIIS配備、API URL差し替え、SPAリライトルールを整備しました。  
さらに、将来のSQL Server移行を見据え、設定で切り替え可能な構成にしています。

### 3. どう運用するか（約4分）

普段の流れは次の通りです。  
開発側はPRで通常テストを通す。  
Jenkinsはノイズ少なめのログで結果を出す。  
不具合時はログから失敗箇所を即特定。  
リリース時はDeploy手順に沿ってバックエンドとフロントを順に配備。  
環境差分はパラメータと設定ファイルで吸収する。  
この運用で、属人化を抑えつつ再現性を確保できます。

### 4. 残課題と次アクション（約3分）

残課題は3つです。  
1つ目、実環境の確定値の反映です。IISサイト名、パス、URL、SQL Server接続情報を最終確定します。  
2つ目、ステージングで通しリハーサルです。初回配備から疎通確認までを一度実施します。  
3つ目、ロールバック確認です。万が一時の戻し手順を実地で確認します。  
ここまで終われば、本番反映のリスクをかなり下げられます。

### 締め

今回の整備で、CIの見通し、起動運用、配備手順の3つが最低限の実運用レベルに乗りました。  
今後は、監視とアラート、リリース判定基準の定義まで広げると、より安定運用に近づきます。

## 開発者向け手順詳細版

### 1. リポジトリ準備

```powershell
git pull --rebase
```

### 2. Jenkinsエージェント自動起動（Windows）

```powershell
cd .\Jenkins
Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process -Force
.\register-scheduled-task.ps1
Get-ScheduledTask -TaskName 'JenkinsAgent-PSWP-Worker1'
```

### 3. バックエンド（PSWPService）IISデプロイ

```powershell
cd .\Deploy
.\deploy-iis.ps1 `
  -SiteName "PSWP" `
  -AppPool "PSWPPool" `
  -DeployPath "C:\inetpub\wwwroot\pswp" `
  -SitePort "80" `
  -SqlServer "localhost" `
  -SqlDatabase "PSWP" `
  -CorsOrigins "https://pswp.example.com"
```

必要に応じて:

```powershell
.\deploy-iis.ps1 -SkipDbSetup
```

### 4. フロントエンド（PSWPFront）IISデプロイ

```powershell
cd .\Deploy
.\deploy-frontend.ps1 `
  -SiteName "PSWPFront" `
  -AppPool "PSWPFrontPool" `
  -DeployPath "C:\inetpub\wwwroot\pswp-front" `
  -SitePort "3000" `
  -ApiUrl "https://api.pswp.example.com"
```

### 5. 動作確認

```powershell
Invoke-WebRequest http://localhost/swagger
Start-Process http://localhost:3000
Get-Website | Where-Object { $_.Name -like "PSWP*" } | Select-Object Name, State, PhysicalPath
```

### 6. トラブル時の切り分け

- バックエンドが起動しない: IISサイト状態、アプリプール、PSWPService配下のappsettings.Production.jsonを確認
- フロント画面は出るがAPIエラー: PSWPFrontのVITE_API_URL値、CORS設定、API URL疎通を確認
- Jenkinsジョブが流れない: ノード状態（Online）、タスクスケジューラ登録状態を確認

### 7. ロールバック（最小）

1. 対象IISサイトを停止
2. 直前バックアップをデプロイ先へ復元
3. IISサイトを再起動
4. Swaggerと画面トップで疎通確認
